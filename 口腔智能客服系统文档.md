## 一、系统概述

### 1.1 系统简介

这是一个基于 Flask 的智能客服 API 系统，主要服务于口腔/医美行业，提供：
- 客户分析与成交概率预测
- 医院排名与推荐
- 流失预警
- 医生/排班/价格查询

### 1.2 技术栈

| 组件 | 技术 | 说明 |
|------|------|------|
| 后端框架 | Flask | Python Web 框架 |
| 数据库 | MySQL | 两个数据库：hospital_db + kfsyscb |
| 连接池 | DBUtils | 数据库连接池管理 |
| 前端 | HTML + CSS + JS | 单页面聊天界面 |
| AI 模型 | Qwen (可选) | 用于生成自然语言回复 |

### 2.2 功能详解

#### 2.2.1 流失预警 (churn_warning)

**功能说明**: 识别有流失风险的客户，按风险等级分类展示

**风险等级定义**:
| 等级 | 颜色 | 条件 | 说明 |
|------|------|------|------|
| critical | 红色 | 派单后 >14天 未跟进 | 紧急，需立即处理 |
| high | 橙色 | 派单后 7-14天 未跟进 | 高风险 |
| medium | 黄色 | 派单后 3-7天 未跟进 | 中风险 |
| low | 蓝色 | 高意向但 ≤3天 未跟进 | 低风险，需关注 |

**触发示例**:
```
"查看流失预警"
"紧急客户有哪些"
"哪些客户没跟进"
"高风险客户"
```

**代码位置**: `dental_chat_api.py` 第 775-953 行

---

#### 2.2.2 医院派单排名 (hospital_ranking)

**功能说明**: 查询指定地区的医院派单量排名

**触发示例**:
```
"上海派单前十医院"
"北京哪些医院成交多"
"广州医院排名"
```

**返回数据**: 排名、医院名称、医院类型、派单数量

**代码位置**: `dental_chat_api.py` 第 705-773 行

---

#### 2.2.3 客户分析 (customer_conversion)

**功能说明**: 综合分析客户信息，预测成交概率

**返回内容**:
- 基本信息（姓名、手机[脱敏]、意向项目、意向地区、来源渠道）
- 成交概率与意向等级（A/B/C/D）
- 同类客户转化率参考
- 跟进建议
- 历史跟进记录

**意向等级定义**:
| 等级 | 概率范围 | 说明 | 建议策略 |
|------|----------|------|----------|
| A | ≥70% | 高意向 | 重点跟进，争取到院 |
| B | 50-69% | 较高意向 | 积极跟进，解决疑虑 |
| C | 30-49% | 低意向 | 轻度维护，发送资讯 |
| D | <30% | 无意向 | 放入公海或长期培养 |

**触发示例**:
```
"分析一下客户4333366"
"查询客户ID 4333366 的成交概率"
"客户4333366的意向怎么样"
```

**代码位置**: `dental_chat_api.py` 第 1361-1595 行

## 八、常见问题

### Q1: 查询客户提示"未找到客户"

**原因**: SQL 查询中可能包含数据库不存在的字段

**解决**: 检查 `query_customer_info()` 函数的 SQL 字段是否与数据库一致

---

### Q2: 项目/地区显示为"未知"

**原因**: linkage 表中没有对应的 ID 记录

**解决**: 检查 `PlasticsIntention` 或 `zx_District` 的值是否在 `un_linkage` 表中存在

---

### Q3: 流失预警没有数据

**原因**: 可能没有符合条件的客户

**检查 SQL**:
```sql
SELECT COUNT(*) FROM un_channel_client
WHERE Status = 0
  AND client_status IN (1,2,3)
  AND EditTime < UNIX_TIMESTAMP(DATE_SUB(NOW(), INTERVAL 3 DAY));
```
