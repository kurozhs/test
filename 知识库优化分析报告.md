## 二、业务场景需求分析

### 2.1 场景一：客服查询成交好的医院

**用户故事**:
> "客服小王想知道最近哪些医院成交率高，方便给顾客推荐靠谱的医院"

**当前系统能力**: ❌ 无法满足

**需要的数据**:
```sql
-- 需要的查询能力
SELECT
    h.hospital_name,
    COUNT(o.id) as order_count,           -- 成交数量
    SUM(o.amount) as total_amount,        -- 成交金额
    AVG(o.amount) as avg_amount,          -- 客单价
    COUNT(o.id) / COUNT(d.id) as conversion_rate  -- 转化率
FROM hospitals h
LEFT JOIN orders o ON h.id = o.hospital_id
LEFT JOIN dispatches d ON h.id = d.hospital_id
WHERE o.status = 'completed'
  AND o.created_at >= DATE_SUB(NOW(), INTERVAL 30 DAY)
GROUP BY h.id
ORDER BY conversion_rate DESC, order_count DESC
```

```python
# 基于经纬度的距离计算
def get_nearby_hospitals(customer_lat, customer_lng, radius_km=5):
    """
    计算顾客位置附近的医院
    使用 Haversine 公式计算球面距离
    """
    sql = """
        SELECT h.*,
            (6371 * acos(
                cos(radians(%s)) * cos(radians(h.latitude)) *
                cos(radians(h.longitude) - radians(%s)) +
                sin(radians(%s)) * sin(radians(h.latitude))
            )) AS distance_km
        FROM hospitals h
        WHERE h.status = 1
        HAVING distance_km <= %s
        ORDER BY distance_km ASC
    """
```

### 4.2 综合评分算法

```python
def calculate_hospital_score(hospital, customer_context):
    """
    医院综合评分算法

    Args:
        hospital: 医院数据
        customer_context: 顾客上下文（位置、预算、意向等）

    Returns:
        综合评分 0-100
    """
    score = 0

    # 1. 成交率评分 (40%)
    conversion_rate = hospital['conversion_rate_30d'] or 0
    conversion_score = min(conversion_rate * 100, 100) * 0.4
    score += conversion_score

    # 2. 距离评分 (30%)
    if customer_context.get('latitude') and hospital.get('latitude'):
        distance = calculate_distance(
            customer_context['latitude'], customer_context['longitude'],
            hospital['latitude'], hospital['longitude']
        )
        # 5km内满分，超过20km为0分
        distance_score = max(0, (20 - distance) / 20 * 100) * 0.3
        score += distance_score

    # 3. 价格匹配度 (20%)
    if customer_context.get('budget_max'):
        avg_price = hospital.get('avg_order_amount', 0)
        budget = customer_context['budget_max']
        if avg_price <= budget:
            price_score = 100 * 0.2
        else:
            price_score = max(0, (1 - (avg_price - budget) / budget)) * 100 * 0.2
        score += price_score

    # 4. 满意度评分 (10%)
    satisfaction = hospital.get('avg_satisfaction', 0)
    satisfaction_score = satisfaction * 20 * 0.1  # 5分制转100分
    score += satisfaction_score

    return round(score, 2)
```

## 七、讨论要点

### 需要确认的问题

1. **数据来源**
   - 派单和成交数据目前在哪个系统？
   - 是否有现成的API可以对接？
   - 数据更新频率要求？（实时/小时/天）

2. **顾客数据**
   - CRM系统是什么？
   - 跟进记录的数据结构是怎样的？
   - 是否有顾客位置信息？

3. **业务规则**
   - 成交率的计算口径是什么？（30天/90天/全部）
   - 医院排名的维度有哪些？
   - 话术推荐的触发机制？

4. **技术选型**
   - 是否需要引入向量数据库做语义检索？
   - 是否需要实时计算还是预计算？
   - 距离计算是否需要考虑路线规划？
