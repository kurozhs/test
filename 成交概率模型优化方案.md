## 一、项目背景

### 1.1 现有系统简介

口腔智能客服系统包含一个**客户成交概率分析模块**，用于：
- 预测客户成交可能性
- 为客服提供跟进优先级建议
- 生成个性化跟进话术

### 1.2 为什么需要优化

| 问题类别 | 具体问题 | 业务影响 |
|----------|----------|----------|
| 漏斗缺失 | 未识别客户所处阶段 | "已派单"和"刚注册"可能算出相同分数 |
| 时间盲区 | 仅判断7天内有无活动 | 60天前注册的沉默客户被高估 |
| 信号粗糙 | "太贵"被当作负面信号 | 实际是议价信号，误判高意向客户 |
| 分级太粗 | 仅ABCD四级 | 客服无法区分轻重缓急 |

---

## 二、优化目标

### 2.1 核心目标

```
将【静态评分模型】升级为【漏斗驱动 + 时间衰减 + 智能信号】的动态模型
```

### 2.2 量化指标

| 指标 | 当前值 | 目标值 | 说明 |
|------|--------|--------|------|
| A级客户实际转化率 | 约15% | >25% | 高意向识别更准 |
| D级客户误判率 | 约20% | <10% | 减少放弃有效客户 |
| 客服人均跟进效率 | 基线 | +20% | 优先级更清晰 |

### 2.3 新旧模型对比

```
旧模型:
总分 = 客户状态(30%) + 跟进活跃度(25%) + 意向信号(25%) + 相似转化率(20%)
等级 = A(≥70) / B(50-70) / C(30-50) / D(<30)

新模型:
总分 = 漏斗基础分 + 时间调整 + 信号调整 + 历史校准
等级 = A1/A2/B1/B2/C1/C2/D (7级)
```

### 任务2: 实现漏斗阶段检测函数

**任务描述**: 根据客户数据自动识别当前所处的转化漏斗阶段

**所属文件**: `conversion_model_v2.py`

**函数签名**:

```python
def detect_funnel_stage(
    customer: Dict,
    follow_records: List[Dict],
    hospital_orders: List[Dict],
    appointments: List[Dict],
    orders: List[Dict]
) -> Tuple[str, int]:
    """
    检测客户当前所处的漏斗阶段

    Args:
        customer: 客户基本信息字典，包含 Status, Client_Id 等
        follow_records: 跟进记录列表
        hospital_orders: 医院派单记录列表
        appointments: 预约记录列表
        orders: 成交记录列表

    Returns:
        Tuple[str, int]: (阶段名称, 基础分)

    阶段说明:
        - registered (10分): 刚注册，无任何互动
        - consulted (20分): 已有跟进记录
        - need_confirmed (35分): 跟进中有项目讨论
        - hospital_matched (50分): 已推荐/匹配医院
        - dispatched (65分): 已派单到医院
        - appointed (80分): 已预约时间
        - visited (90分): 已到院
        - converted (100分): 已成交
    """
```

**实现逻辑**:

```python
def detect_funnel_stage(customer, follow_records, hospital_orders, appointments, orders):
    """检测客户漏斗阶段"""
    from signal_dictionary import FUNNEL_STAGE_SCORES

    # 1. 已成交 - 最高优先级
    if customer.get('Status') == 1 or orders:
        return 'converted', FUNNEL_STAGE_SCORES['converted']

    # 2. 已到院 - 检查派单状态
    for order in (hospital_orders or []):
        # arrive_status: 1=已到院
        if order.get('arrive_status') == 1:
            return 'visited', FUNNEL_STAGE_SCORES['visited']
        # surgery_status: 10=手术完成 (也算到院)
        if order.get('surgery_status') == 10:
            return 'visited', FUNNEL_STAGE_SCORES['visited']

    # 3. 已预约 - 检查有效预约
    for appt in (appointments or []):
        # status: 1=客服待确认, 2=医院待确认, 99=预约成功
        if appt.get('status') in [1, 2, 99]:
            return 'appointed', FUNNEL_STAGE_SCORES['appointed']

    # 4. 已派单
    if hospital_orders:
        return 'dispatched', FUNNEL_STAGE_SCORES['dispatched']

    # 5. 已匹配医院 - 检查跟进记录中是否有医院推荐
    hospital_keywords = ['推荐', '派单', '医院：', '医院:', '已为您', '安排', '对接']
    for record in (follow_records or [])[:10]:  # 只检查最近10条
        content = record.get('content', '') or record.get('Content', '') or ''
        if any(kw in content for kw in hospital_keywords):
            return 'hospital_matched', FUNNEL_STAGE_SCORES['hospital_matched']

    # 6. 需求已确认 - 跟进记录中有项目相关讨论
    project_keywords = ['种植', '矫正', '正畸', '牙齿', '方案', '价格', '多少钱',
                        '洗牙', '补牙', '拔牙', '美白', '全瓷', '烤瓷']
    for record in (follow_records or [])[:10]:
        content = record.get('content', '') or record.get('Content', '') or ''
        if any(kw in content for kw in project_keywords):
            return 'need_confirmed', FUNNEL_STAGE_SCORES['need_confirmed']

    # 7. 已咨询 - 有跟进记录
    if follow_records:
        return 'consulted', FUNNEL_STAGE_SCORES['consulted']

    # 8. 刚注册 - 默认状态
    return 'registered', FUNNEL_STAGE_SCORES['registered']

from datetime import datetime
from signal_dictionary import TIME_DECAY_CONFIG

def calculate_time_decay(customer, follow_records):
    """计算时间衰减"""
    now = datetime.now()
    details = {
        'days_since_register': None,
        'days_since_last_follow': None,
        'register_adjustment': 0,
        'follow_adjustment': 0,
    }

    # 1. 计算注册天数
    register_time = customer.get('RegisterTime')
    if register_time:
        try:
            if isinstance(register_time, (int, float)):
                register_dt = datetime.fromtimestamp(register_time)
            else:
                register_dt = register_time
            days_since_register = (now - register_dt).days
            details['days_since_register'] = days_since_register

            # 根据配置计算调整分
            for (min_days, max_days), adjustment in TIME_DECAY_CONFIG['register_days'].items():
                if min_days <= days_since_register < max_days:
                    details['register_adjustment'] = adjustment
                    break
        except Exception:
            pass

    # 2. 计算跟进间隔
    if follow_records:
        latest_record = follow_records[0]
        last_follow_time = (latest_record.get('time') or
                           latest_record.get('CMPostTime') or
                           latest_record.get('Time'))
        if last_follow_time:
            try:
                if isinstance(last_follow_time, (int, float)):
                    last_follow_dt = datetime.fromtimestamp(last_follow_time)
                else:
                    last_follow_dt = last_follow_time
                days_since_follow = (now - last_follow_dt).days
                details['days_since_last_follow'] = days_since_follow

                # 根据配置计算调整分
                for (min_days, max_days), adjustment in TIME_DECAY_CONFIG['follow_gap_days'].items():
                    if min_days <= days_since_follow < max_days:
                        details['follow_adjustment'] = adjustment
                        break
            except Exception:
                pass
    else:
        # 无跟进记录，视为超过30天
        details['days_since_last_follow'] = 999
        details['follow_adjustment'] = -15

    # 3. 计算总调整分 (限制在 -30 ~ +20)
    total_adjustment = details['register_adjustment'] + details['follow_adjustment']
    total_adjustment = max(-30, min(20, total_adjustment))

    return total_adjustment, details
```

```python
import re
from datetime import datetime
from signal_dictionary import SIGNAL_DICTIONARY

def analyze_intent_signals(follow_records):
    """分析意向信号"""
    now = datetime.now()

    details = {
        'hot_count': 0,
        'hot_score': 0,
        'negotiation_count': 0,
        'negotiation_score': 0,
        'concern_count': 0,
        'concern_score': 0,
        'negative_count': 0,
        'negative_score': 0,
        'timing_count': 0,
        'timing_score': 0,
        'signal_details': [],
    }

    for record in (follow_records or []):
        content = record.get('content', '') or record.get('Content', '') or record.get('CMContent', '') or ''
        if not content:
            continue

        # 获取记录时间，计算时间权重
        record_time = (record.get('time') or
                       record.get('CMPostTime') or
                       record.get('Time'))

        time_weight = 1.0
        if record_time:
            try:
                if isinstance(record_time, (int, float)):
                    record_dt = datetime.fromtimestamp(record_time)
                else:
                    record_dt = record_time
                days_ago = (now - record_dt).days
                # 时间权重：最近的权重高，最低0.3
                time_weight = max(0.3, 1 - days_ago * 0.02)
            except Exception:
                pass

        # 遍历各类信号
        for signal_type, patterns in SIGNAL_DICTIONARY.items():
            for pattern, base_score in patterns:
                try:
                    if re.search(pattern, content, re.IGNORECASE):
                        weighted_score = base_score * time_weight

                        # 更新统计
                        count_key = f'{signal_type}_count'
                        score_key = f'{signal_type}_score'
                        if count_key in details:
                            details[count_key] += 1
                            details[score_key] += weighted_score

                        # 记录详情
                        details['signal_details'].append({
                            'type': signal_type,
                            'pattern': pattern,
                            'base_score': base_score,
                            'time_weight': round(time_weight, 2),
                            'weighted_score': round(weighted_score, 2),
                        })

                        # 每条记录每类信号只匹配一次（避免重复计分）
                        break
                except re.error:
                    # 正则表达式错误，当作普通字符串匹配
                    if pattern in content:
                        weighted_score = base_score * time_weight
                        count_key = f'{signal_type}_count'
                        score_key = f'{signal_type}_score'
                        if count_key in details:
                            details[count_key] += 1
                            details[score_key] += weighted_score
                        break

    # 计算总调整分
    # 高意向和议价是正面的，顾虑是轻度负面，负面是重度负面
    adjustment = 0
    adjustment += min(details['hot_score'], 15)           # 高意向信号，最多+15
    adjustment += min(details['negotiation_score'], 10)   # 议价信号，最多+10
    adjustment -= min(details['concern_score'], 5)        # 顾虑信号，最多-5
    adjustment -= min(details['negative_score'], 20)      # 负面信号，最多-20
    # timing信号不直接影响分数，但会影响话术推荐

    # 限制范围
    adjustment = max(-20, min(20, int(adjustment)))

    return adjustment, details
```

**验收标准**:
- [ ] 正确匹配5类信号
- [ ] 时间权重计算正确
- [ ] 调整分在 -20 ~ +20 范围内
- [ ] 返回详细的匹配信息
- [ ] 正则表达式异常处理

        similar_rate_data: 相似客户转化率数据，来自 query_similar_conversion_rate()

```python
def calibrate_with_history(current_score, similar_rate_data):
    """历史数据校准"""
    details = {
        'calibration_applied': False,
        'reference_rate': None,
        'sample_size': None,
        'confidence': 'insufficient',
    }

    if not similar_rate_data:
        return 0, details

    # 优先使用精准匹配数据
    precise = similar_rate_data.get('precise', {})
    overall = similar_rate_data.get('overall', {})

    reference_data = None
    if precise.get('total', 0) >= 30:
        reference_data = precise
        details['confidence'] = 'high'
    elif precise.get('total', 0) >= 10:
        reference_data = precise
        details['confidence'] = 'medium'
    elif overall.get('total', 0) >= 50:
        reference_data = overall
        details['confidence'] = 'low'

    if not reference_data:
        return 0, details

    # 计算历史转化率对应的分数
    historical_rate = reference_data.get('rate', 0)
    historical_score = historical_rate * 100  # 转化率转为百分制
    sample_size = reference_data.get('total', 0)

    details['calibration_applied'] = True
    details['reference_rate'] = round(historical_rate * 100, 1)
    details['sample_size'] = sample_size

    # 校准逻辑：当前分数向历史数据靠拢
    # 置信度越高，靠拢程度越大
    confidence_weight = {
        'high': 0.3,
        'medium': 0.2,
        'low': 0.1,
    }
    weight = confidence_weight.get(details['confidence'], 0)

    adjustment = (historical_score - current_score) * weight
    adjustment = max(-10, min(10, int(adjustment)))

    details['adjustment_reason'] = (
        f"历史转化率{details['reference_rate']}%，"
        f"样本{sample_size}，"
        f"{'上调' if adjustment > 0 else '下调' if adjustment < 0 else '不调整'}{abs(adjustment)}分"
    )

    return adjustment, details
```

- [ ] 正确处理精准匹配和整体匹配数据
- [ ] 置信度判断正确
- [ ] 调整分在 -10 ~ +10 范围内
- [ ] 返回校准依据详情

    customer, follow_records, hospital_orders, appointments, orders, similar_rate_data=None

    if signal_details.get('negotiation_count', 0) > 0:
        # 有议价信号 → 优惠话术
        if 'limited_offer' in level_templates:
            template = level_templates['limited_offer']
            scripts.append({
                'scenario': template['scenario'],
                'script': template['template'].format(
                    customer_name=customer_name,
                    project=context.get('project', '项目'),
                    discount=context.get('discount', '优惠'),
                    end_date=context.get('end_date', '本月底'),
                ),
                'priority': 2,
                'reason': '检测到议价信号，推送优惠',
            })

    # 3. 补充该等级的通用话术
    for template_key, template in level_templates.items():
        # 避免重复
        if any(s['scenario'] == template['scenario'] for s in scripts):
            continue
        scripts.append({
            'scenario': template['scenario'],
            'script': template['template'].format(
                customer_name=customer_name,
                project=context.get('project', '项目'),
                hospital=context.get('hospital_name', '医院'),
                **context
            ),
            'priority': 3,
            'reason': f'{level}级客户通用话术',
        })

    # 按优先级排序
    scripts.sort(key=lambda x: x['priority'])

    return scripts[:5]  # 最多返回5条


def get_concern_answer(concern_keyword):
    """根据顾虑关键词返回标准解答"""
    answers = {
        '怕疼': '我们采用进口麻醉，全程无痛，很多顾客做完都说比想象中轻松多了',

    for key, answer in answers.items():

## 六、风险与注意事项

### 6.1 兼容性风险

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| API返回结构变化 | 前端可能报错 | 保持核心字段兼容，新增字段放在新key下 |
| 等级从4级变7级 | 前端显示异常 | 提前通知前端适配，或提供映射函数 |
| 概率分数分布变化 | 客服不适应 | 提供新旧分数对照说明 |

### 6.2 性能风险

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 正则匹配耗时 | 响应变慢 | 预编译正则，限制匹配记录数 |
| 信号词库过大 | 内存占用 | 按类别懒加载 |

### 6.3 回退方案

```python
# 在 dental_chat_api.py 中保留开关
USE_V2_MODEL = os.getenv('USE_V2_MODEL', 'true').lower() == 'true'

if USE_V2_MODEL:
    probability = calculate_conversion_probability_v2(...)
else:
    probability = calculate_conversion_probability(...)  # 旧模型
```
