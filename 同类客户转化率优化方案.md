def query_similar_conversion_rate(intention: int, region: int) -> Dict:

### 1.2 当前逻辑问题

| 问题 | 影响 | 严重程度 |
|------|------|----------|
| 仅匹配意向项目+地区 | 匹配维度单一，样本代表性差 | ⚠️ 中 |
| 固定90天窗口 | 样本量可能过少或过多 | ⚠️ 中 |
| 未考虑来源渠道 | 不同渠道转化率差异大 | 🔴 高 |
| 未考虑客服能力 | 客服水平直接影响转化 | 🔴 高 |
| 未考虑预算匹配 | 预算是成交核心因素 | 🔴 高 |
| 未排除当前客户 | 自己算入样本，数据污染 | ⚠️ 中 |

## 二、资深数据分析师视角

### 2.1 转化率计算模型重构

**核心思路**: 从单一维度匹配 → 多维度加权相似度匹配

```
┌─────────────────────────────────────────────────────────────────┐
│                    客户相似度计算模型                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   相似度 = Σ(维度权重 × 维度匹配度)                              │
│                                                                 │
│   维度1: 意向项目匹配 (权重 30%)                                 │
│         - 完全匹配: 100%                                        │
│         - 同大类: 70%                                           │
│         - 相关类: 40%                                           │
│                                                                 │
│   维度2: 地区匹配 (权重 20%)                                     │
│         - 同区: 100%                                            │
│         - 同市: 70%                                             │
│         - 同省: 40%                                             │
│                                                                 │
│   维度3: 来源渠道匹配 (权重 20%)                                 │
│         - 完全匹配: 100%                                        │
│         - 同类渠道: 60%                                         │
│                                                                 │
│   维度4: 注册时间接近度 (权重 15%)                               │
│         - 同月: 100%                                            │
│         - 相邻月: 70%                                           │
│         - 同季度: 40%                                           │
│                                                                 │
│   维度5: 跟进次数相似 (权重 15%)                                 │
│         - 差值≤2: 100%                                          │
│         - 差值≤5: 70%                                           │
│         - 差值>5: 40%                                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 分层转化率分析

不应该只给一个笼统的转化率，应该分层展示：

```
┌─────────────────────────────────────────────────────────────────┐
│                    分层转化率报告                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【整体转化率】                                                  │
│  ├── 全平台平均转化率: 12.5% (样本 50,000)                      │
│  └── 该项目类型转化率: 15.2% (样本 3,200)                       │
│                                                                 │
│  【精准匹配转化率】                                              │
│  ├── 同项目+同地区: 18.5% (样本 120)                            │
│  ├── 同项目+同渠道: 22.3% (样本 85)                             │
│  └── 同项目+同地区+同渠道: 25.1% (样本 32)                      │
│                                                                 │
│  【时间趋势】                                                    │
│  ├── 近30天转化率: 16.8% ↑                                      │
│  ├── 近60天转化率: 15.2%                                        │
│  └── 近90天转化率: 14.5% ↓                                      │
│                                                                 │
│  【客服维度】                                                    │
│  ├── 当前客服历史转化率: 28.5% (高于平均)                       │
│  └── 该项目类型该客服转化率: 32.1%                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 三、资深数据库管理师视角

### 3.1 当前SQL问题分析

```sql
-- 当前SQL
SELECT COUNT(*), SUM(CASE WHEN Status = 1 THEN 1 ELSE 0 END)
FROM un_channel_client
WHERE PlasticsIntention = %s
  AND (zx_District = %s OR client_region = %s)
  AND RegisterTime >= UNIX_TIMESTAMP(DATE_SUB(NOW(), INTERVAL 90 DAY))
```

**问题诊断**:

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 全表扫描风险 | OR条件导致索引失效 | 拆分查询或使用UNION |
| 无索引覆盖 | 缺少复合索引 | 创建多列索引 |
| 实时计算开销大 | 每次查询都要统计 | 预计算+缓存 |
| 时间函数 | UNIX_TIMESTAMP每次执行 | 参数化传入 |

### 3.2 索引优化建议

```sql
-- 1. 创建复合索引（推荐）
CREATE INDEX idx_client_conversion_analysis
ON un_channel_client (PlasticsIntention, zx_District, RegisterTime, Status);

-- 2. 创建覆盖索引（如果查询频繁）
CREATE INDEX idx_client_region_analysis
ON un_channel_client (client_region, PlasticsIntention, RegisterTime, Status);

-- 3. 查看现有索引
SHOW INDEX FROM un_channel_client;
```

### 3.3 优化后的SQL查询

```sql
-- 优化方案1: 拆分OR条件，使用UNION ALL
SELECT
    COUNT(*) as total_count,
    SUM(CASE WHEN Status = 1 THEN 1 ELSE 0 END) as converted_count,
    AVG(CASE WHEN Status = 1 THEN 1.0 ELSE 0.0 END) as conversion_rate
FROM (
    -- 按意向地区匹配
    SELECT Client_Id, Status FROM un_channel_client
    WHERE PlasticsIntention = ?
      AND zx_District = ?
      AND RegisterTime >= ?
      AND Client_Id != ?  -- 排除当前客户

    UNION

    -- 按地区分类匹配（当意向地区为空时）
    SELECT Client_Id, Status FROM un_channel_client
    WHERE PlasticsIntention = ?
      AND client_region = ?
      AND zx_District = 0
      AND RegisterTime >= ?
      AND Client_Id != ?
) AS similar_clients;
```

### 3.4 预计算表设计（推荐方案）

创建物化视图/统计表，定时更新：

```sql
-- 转化率统计表（每日凌晨更新）
CREATE TABLE conversion_rate_stats (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,

    -- 维度字段
    intention_id INT NOT NULL COMMENT '意向项目ID',
    intention_parent_id INT COMMENT '意向项目父级ID',
    region_id INT COMMENT '意向地区ID',
    region_province_id INT COMMENT '省份ID',
    from_type INT COMMENT '来源类型',
    kf_id INT COMMENT '客服ID',

    -- 时间窗口
    time_window ENUM('7d', '30d', '90d', '180d', 'all') NOT NULL,

    -- 统计数据
    total_count INT DEFAULT 0 COMMENT '样本总数',
    converted_count INT DEFAULT 0 COMMENT '成交数量',
    conversion_rate DECIMAL(5,4) COMMENT '转化率',
    avg_deal_amount DECIMAL(10,2) COMMENT '平均成交金额',
    avg_follow_days INT COMMENT '平均跟进天数',

    -- 元数据
    calculated_at DATETIME DEFAULT NOW(),

    -- 索引
    INDEX idx_lookup (intention_id, region_id, time_window),
    INDEX idx_intention (intention_id, time_window),
    INDEX idx_region (region_id, time_window)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='转化率预计算统计表';
```

### 3.5 定时任务更新脚本

```sql
-- 每日更新转化率统计
INSERT INTO conversion_rate_stats
    (intention_id, region_id, time_window, total_count, converted_count, conversion_rate)
SELECT
    PlasticsIntention,
    zx_District,
    '30d',
    COUNT(*),
    SUM(CASE WHEN Status = 1 THEN 1 ELSE 0 END),
    AVG(CASE WHEN Status = 1 THEN 1.0 ELSE 0.0 END)
FROM un_channel_client
WHERE RegisterTime >= UNIX_TIMESTAMP(DATE_SUB(NOW(), INTERVAL 30 DAY))
  AND PlasticsIntention > 0
GROUP BY PlasticsIntention, zx_District
ON DUPLICATE KEY UPDATE
    total_count = VALUES(total_count),
    converted_count = VALUES(converted_count),
    conversion_rate = VALUES(conversion_rate),
    calculated_at = NOW();
```

## 六、总结

| 维度 | 当前状态 | 优化后 |
|------|----------|--------|
| 匹配维度 | 2个（项目+地区） | 5个（项目+地区+渠道+时间+客服） |
| 置信度 | 无 | 有（高/中/低/不足） |
| 性能 | 实时全表扫描 | 预计算+索引优化 |
| 展示 | 单一数字 | 分层报告 |
| 参考价值 | ⭐⭐ | ⭐⭐⭐⭐⭐ |

**核心改进点**:
1. 从"单一转化率" → "多维度分层分析"
2. 从"无置信度" → "样本量置信度标识"
3. 从"实时计算" → "预计算+缓存"
4. 从"数字展示" → "可视化报告"
