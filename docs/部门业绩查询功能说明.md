# 部门业绩 & 公司业绩查询功能

## 一、功能概述

新增两个意图：`department_stats`（部门业绩）和 `company_stats`（公司整体业绩），用户可以通过自然语言查询各事业群和全公司的成交数据。

**触发示例：**
- "TEG部门业绩" / "BCG上月业绩" / "TEG流水业绩" → `department_stats`
- "公司这个月业绩" / "公司上月成交" → `company_stats`

## 三、核心逻辑（对应 PHP 源码）

### 3.1 两种查询模式

对应 PHP 文件：`/phpcms/modules/tongji/tongji_pay.php` → `show_tongji_detail()` → `get_data_detail()`

| 模式 | SQL条件 | 说明 | 触发词 |
|------|---------|------|--------|
| **审核业绩**（默认） | `checktime BETWEEN ... AND ..., status=1` | 按审核通过时间统计，无 `split_parent` 过滤 | 默认 / "审核" |
| **成交业绩** | `addtime BETWEEN ... AND ..., status!=3, split_parent=0` | 按录入时间统计 | "流水" / "成交时间" / "录入" |

**为什么默认审核模式？** PHP 统计页面 `tongji_pay.php` line 3481 默认 `timetype = 'checktime'`，用户日常看的就是审核业绩。

### 3.2 审核时间范围

审核模式下，"本月"/"上月"不是自然月，而是从 `un_hospital_refund_time` 表取的审核周期。

```
例：3月审核周期 = 1月22日 ~ 3月2日（不是3/1~3/31）
```

函数：`_get_audit_time_range(year, month)` → 查 `un_hospital_refund_time` 表

### 3.3 部门渠道映射

```
un_admin_department.channel_id → _parse_php_array() → channel_id 列表
```

**注意：** 数据库里存的是 PHP `var_export` 格式：
```php
array (
  0 => '5701',
  1 => '5704',
  ...
)
```
不是 `array(5701,5704,...)`，`_parse_php_array()` 两种格式都能解析。

**老渠道兼容（reflect）：** `_get_reflect_channel_ids()` 硬编码了 `app_data_config.php` 里的 `reflect` 映射，把老 channel_id 也加入查询范围。如果 `app_data_config.php` 的 reflect 配置变了，需要同步修改这个函数。

### 3.4 币种转换

对应 PHP：`tongji_pay.php` line 3560-3569

```
优先用 checknumber（审核金额，已是人民币），checknumber != 0 时直接用
否则按 currency_type 转换：
  1=韩币 → number / 动态汇率（从 un_korea_rate 表取，当前约212）
  3=美元 → number / 0.14
  4=泰铢 → number / 4.6
  5=日元 → number / 15
  2=人民币 → 直接用 number
```

**韩币汇率是动态的！** 函数 `_get_korea_rate(addtime)` 按月查 `un_korea_rate` 表，查不到回退上月，最终兜底 160。

### 3.5 项目分类

对应 PHP：`department_statistics.class.php` line 244-276

```
1. verym_project 匹配口腔(5707子项) → kq
2. verym_project 匹配眼科(7552子项) → yk
3. 以上都不是 + currency_type=2(人民币) → ym（医美）
4. 以上都不是 + currency_type=1(韩币) → kr（韩国）
5. 其他币种（美元/泰铢/日元） → other（只计入总计，不归入分项）
```

口腔/眼科的子项目ID从 `un_linkage` 表查：`linkageid=5707`(口腔) 和 `linkageid=7552`(眼科) 的 `arrchildid` 字段。

## 四、防误触发

| 场景 | 处理 |
|------|------|
| "TEG渠道注册情况" | 有"渠道"+"注册"关键词 → 仍走 `source_analysis`，不会命中 `department_stats` |

## 五、如何修改/扩展

### 5.1 新增部门

在 `DEPARTMENT_MAP` 里加一行：
```python
"XXX": (部门ID, "XXX-部门中文名"),
```
部门ID 是 `un_admin_department` 表的 `id` 字段。

### 5.2 修改审核时间范围逻辑

改 `_get_audit_time_range()` 函数，它查的是 `un_hospital_refund_time` 表。

### 5.3 修改币种汇率

韩币汇率自动从 `un_korea_rate` 表取，其他币种（美元/泰铢/日元）在 `_convert_currency_to_rmb()` 里硬编码。

### 5.4 修改 reflect 老渠道映射

在 `_get_reflect_channel_ids()` 里改硬编码，对应 PHP 的 `app_data_config.php` → `reflect` 配置。

### 5.5 调整默认查询模式

在 `smart_query()` 路由里改 `mode` 判断逻辑。当前规则：
- 含"流水"/"成交时间"/"录入" → `addtime` 模式
- 其他 → `checktime` 模式（审核，默认）

## 六、调试方法

### 直接调API测试
```bash
curl -X POST http://localhost:8090/api/dental/chat \
  -H "Content-Type: application/json" \
  -d '{"question": "teg部门上个月业绩多少"}'
```

### 查看意图识别结果
返回 JSON 里的 `data.department_stats` 包含完整查询结果。

### 对数
和 PHP 系统对数时注意：
1. 确认对方看的是**审核业绩还是成交业绩**（默认审核）
2. 确认对方的**时间范围**（审核模式下不是自然月，是从 `hospital_refund_time` 表取的）
3. 韩币汇率是否一致（查 `un_korea_rate` 表当月数据）
